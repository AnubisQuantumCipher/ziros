## What belongs here

Release documentation in this directory should be:

- source-backed
- lane-honest
- explicit about what changed
- explicit about what was verified
- explicit about what remains provisional, delegated, or metadata-only

This is not the place for raw local planning notes, unreviewed scratch files, or unverifiable marketing copy.
