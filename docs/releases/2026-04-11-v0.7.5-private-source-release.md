# ZirOS Private Source Release — v0.7.5

Status: private source release  
Repository visibility: private  
Release tag: `v0.7.5`  
Release date: 2026-04-11

## Summary

This release cuts a fresh private source snapshot after `v0.7.4` and closes the remaining trade-finance Midnight packaging drift that was still preventing the emitted subsystem bundle from behaving like a self-consistent operator artifact.

The dominant change in `v0.7.5` is not a new proving primitive. It is release discipline:

- the trade-finance Midnight validator now works both from the showcase artifact root and from the packaged subsystem root
- per-contract and per-call prepare artifacts are preserved instead of being overwritten by the last loop iteration
- the packaged subsystem no longer ships repo-dependent proof/materialization scripts as if they were runnable in isolation
- the release metadata, operator readiness docs, and public attestation inputs are refreshed to the current version and commit

## Highlights

### 1. Trade-finance Midnight package closure

The trade-finance subsystem release surface is now materially cleaner and more honest.

Key changes:

- `scripts/validate_private_trade_finance_midnight_contracts.sh` detects showcase versus subsystem layouts
- validator binary selection now refuses a `zkf` binary that does not actually expose `midnight`
- deploy-prepare and call-prepare auxiliary assets are preserved under per-contract and per-call asset roots
- validation summary/report JSON stays on the same schema family and adds the new asset-root fields additively

This matters because the shipped package is supposed to be an operator surface, not a fragile one-shot export whose prepared artifacts collapse on reuse.

### 2. Public-package honesty for shipped scripts

The packaged subsystem now fails closed where it should fail closed.

Specifically, the shipped Lean, Rocq, Verus, and materialize scripts now:

- exit with code `64`
- explain that the public package does not ship theorem-source or repo helper tooling
- point the operator to the shipped formal logs and certificates instead

That is the correct contract for a public evidence bundle.

### 3. Refreshed trade-finance forensics

This release adds and refreshes source-backed trade-finance forensic artifacts:

## Boundaries

- This remains a **private source** release, not a public-source cutover.
- Midnight stays an operator-grade contract lane with honest environment boundaries; it is not reclassified as a native strict recursive proof lane.
- The public attestation repo is refreshed from the private release inputs, but that does not make the private workspace itself public.
- Uncommitted work in the original private checkout is intentionally excluded from the clean release branch used for this release cut.

## Why v0.7.5 exists

`v0.7.4` already exists as a tagged release. The current checkout includes additional committed work after that tag, including the trade-finance package closure. Cutting `v0.7.5` is the honest way to publish a coherent next snapshot instead of silently rewriting prior release metadata.
