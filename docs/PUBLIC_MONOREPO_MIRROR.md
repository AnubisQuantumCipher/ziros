# ZirOS Public Monorepo Mirror

This document defines the first public-safe open-core mirror for:

- `AnubisQuantumCipher/ziros`

## Release Posture

The first public cut is intentionally:

- staged first, not an in-place cutover
- open core only
- Apache-2.0 for included crates
- source-visible and clone-root buildable for the included workspace slice

It is not a claim that the current private staging workspace can be published as
one piece without rewriting boundaries. The public repository is the
authoritative release surface for the included open-core tranche only.

## First-Cut Boundary

The staged mirror keeps the same explicit product boundary as the reentry
flagship:

- ground-side mission assurance
- NASA Class D ground-support mission-ops assurance
- normalized-export-based ingestion
- no native replacement claim for GMAT, SPICE, Dymos/OpenMDAO, Trick/JEOD, Basilisk, cFS, or F Prime

Any mission that wants to use ZirOS outputs inside a NASA Class C or higher
decision chain must perform an independent assessment outside ZirOS.

## Included In The First Public Cut

The first public mirror is built around the open-core theorem-first stack needed
for the accepted reentry mission-assurance flow and the shared aerospace kit.

The staged mirror is generated by:

- `scripts/stage_ziros_public_mirror.py`

That staging tool:

- computes the open-core workspace closure from the reentry operator surface
- excludes private or BSL-only crates from the first cut
- rewrites the staged workspace manifests to a CPU-first open-core posture
- emits public mirror docs, release scaffolding, SBOM metadata, and provenance metadata

## Validation Status

As of March 29, 2026, the public mirror cut has been validated as a real
clone-root build surface on the release Apple Silicon host by running:

- `python3 scripts/stage_ziros_public_mirror.py --out target/ziros-public-mirror`
- `cargo build --workspace` from `target/ziros-public-mirror`

That validation matters because it proves the first public cut is not just a
documentation posture. The published tree actually detaches from the private
workspace and builds as an Apache-2.0-safe open-core slice without hidden local
path dependencies.

## Explicit Exclusions

The first public cut does not include:

- `zkf-api`
- `zkf-backends-pro`
- `zkf-metal`
- unrelated private staging surfaces outside the theorem-first open-core reentry tranche

The purpose of this exclusion list is honesty, not caution theater. The first
public mirror should not pretend excluded private or BSL-only crates are part of
the same open-core trust story.

## Required Mirror Release Surfaces

The staged mirror must carry:

- `release/sbom/open_core_workspace_sbom.json`
- `release/provenance/public_mirror_provenance.json`
- `release/theorem_coverage_status.json`
- `release/public_bundle/`
- `release/private_bundle/`
- a candid long-form flagship report when one exists for the staged tranche

## Flagship Repos

Flagship release repos should become thin product repos pinned to public
`ziros` tags. They should not evolve into divergent code forks.
