# Security

The canonical security model for this repository lives in [docs/SECURITY.md](docs/SECURITY.md).

Use that document for:

- threat model and trusted computing base
- formal-verification scope boundaries
- Groth16 setup and trusted-setup trust-lane notes
- verified Metal boundary language

Public showcase bundles must keep their claims aligned with the evidence shipped in the bundle itself. If a bundle is labeled `demo-only`, it is not release-safe proof-bearing material.
